MySql数据类型和表的约束

发表于 2018-06-12 | 分类于 数据库 ， MySql | 热度: ℃
字数统计: 2,111 | 阅读时长 ≈ 8

此篇博客主要的内容是关于MySql数据库的数据类型进行介绍。顺带将一下表的约束。

大致MySql数据类型内容就分为上面的的这些这里我们挑其中的重点来说。

数据类型

数值类型

整数

  1. tinyint

在这里说一下使用注意事项，它能表示的范围在-128~128之间，如果超过这个范围插入会报错的，但是5.1.7版本中虽然不会报错，但是会截取为最接近的合法值。
注意：1.在mysql中，整形可以指定有无符号，默认是有符号的；
2.可以通过UNSIGNED来说明某个字段是无符号的

关于数值类型的使用大致是一样的下面自己去以此类推。在使用的时候合理的选择数据类型

  • 在这里强调一点那就是unsignesd使用的注意事项：

如果你create table tt1(a int unsigned ,b int unsigned)，然后分别insert into tt1 values(1,2);然后让select a-b from tt1;这个时候你会发现结果并非你所想的-1，而是一串很大的数字！但是他们在底层的存储都是一样的，只是看待的方式不一样而已。所以可以通过下面的方法来改变。

所以一般在项目中不建议使用unsigned

  1. bit的使用

位字段类型，M表示每个值的位数,范围从1~64，如果M被忽略则默认为1。一般这种类型按数值的ascii值来显示.

小数

  1. float

m指定显示长度，n指定小数位数，占用四个字节，unsigned表示小数是否有无符号。

注意:如果定义的是float(4,2) unsigned 这时，因为把它指定为无符号的数，范围是 0 ~ 99.99

  1. decimal

定点数m指定长度，d表示小数的位数

  • decimal(5,2)表示的范围是-999.99~999.99
  • decimal(5,2) unsigned表示的范围是0-999.99

decimal和float很像，但是它比float精度高

  • decimal整数最大位数m为65。支持小数最大位数n是30。如果n被省略，默认为0.如果m被省略，默认是10。总的来说就是float精度不够。

字符串

  1. char和varchar
  • char(L): 固定长度字符串，L是可以存储的长度，单位为字符，最大长度值可以为255这里要注意是字符不是字节，即中文英文都可以，但是char的L是多大他就占用多大的空间，即时你没有用那么多空间。
  • varchar(L): 可变长度字符串，L表示字符长度，最大长度65535个字节。他是你用多少它占多少空间

关于varchar(len),len到底是多大，这个len值，和表的编码密切相关：
varchar长度可以指定为0到65535之间的值，但是有1 - 3 个字节用于记录数据大小，所以说有效字节数是65532。当我们的表的编码是utf8时，varchar(n)n最大值是65532/3=21844[因为utf中，一个汉字占用3个字节]，如果编码是gbk，varchar(n)n最大是65532/2=32766（因为gbk中，一个汉字占用2字节）。但注意一行长度不能超过65535

日期和时间类型

这里只说明常用的类型，常用的时间类型如下

  • datetime 时间日期格式 ‘yyyy-mm-d HH:ii:ss’ 表示范围从1000到9999，占用八字节
  • date:日期 ‘yyyy-mm-dd’，占用三字节
  • timestamp：时间戳，从1970年开始的 yyyy-mm-dd HH:ii:ss 格式和datetime完全一致，占用四字节

枚举和Set类型

  1. set集合:对于一个字段可以有多个值的字段类型可以设置为set

    该设定只是提供了若干个选项的值，最终一个单元格中，实际只存储了其中一个值；而且，处于效率考虑，这些值
    实际存储的是“数字”，因为这些选项，每个选项值，一次对应如下数字：1,2,3,….最多65535个；当我们添加枚举值时，也可以添加对应的数字编号

  2. enum枚举：对于一个字段多个中只能选一个。

    该设定只是提供了若干个选项的值，最终一个单元格中，设计可存储了任何其中的多个值；而且，处于效率考虑，这些实际存储的是“数字”，因为这些选项，每个选项值，一次对应如下数字：1,2,4,8,16,32，….最多64个；[偶数]
