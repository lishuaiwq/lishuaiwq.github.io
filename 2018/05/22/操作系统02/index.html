Linux中du和df两个命令的区别

在理解这两个命令之前需要对Linux文件存储有一个大概的认识,即inode，硬盘扇区，‘块’大小，在这里我就不啰嗦了，给出一篇解释比较好的博客，大家自己去了解。

  • https://www.cnblogs.com/xiexj/p/7214502.html

du

du的英文为:disk usage,含义是磁盘空间使用情况，功能是逐级进入指定目录的每一个子目录并显示该目录占用文件系统数据块的情况，如果没有指定目录，则对当前的目录进行统计。

  • du -h 使用kb的方式显示

  • du -s 只显示当前目录大小的总和

个人理解注意统计当前目录的话，只是按其中的文件目录个数来统计大小的

  • 扩充一下 ls -l 中的total 的意义，其特是统计当前显示出来的文件和目录所占的大小，一般来说也是按个数统计。

df

于du不同的是，du是面向文件的命令，只计算被文件占用的空间。不计算文件系统metadata 占用的空间。df则是基于文件系统总体来计算，通过文件系统中未分配空间来确定系统中已经分配空间的大小。df命令可以获取硬盘占用了多少空间，还剩下多少空间，它也可以显示所有文件系统对i节点和磁盘块的使用情况。

  • df -i 查看inode使用情况
  • fd -a 显示全部磁盘系统档案的使用情况

Linux常用系统命令

  • 查看Linux cpu的大小
    • cat /proc/cpuinfo因为在这个文件中有cpu的详细信息
  • 查看Linux 内存大小
    • cat /proc/meminfo | grep MemTotal
  • 查看硬盘大小
    • fdisk -l | grep Disk
  • uname -a 查看内核/操作系统/cpu的信息
  • 查看操作系统的版本信息
    • head -n -1 /etc/issue
  • 查看Linux计算机名字
    • hostname
  • 查看环境变量资源
    • env
  • 查看内存使用量和交换区使用量
    • free -m
  • 查看各分区的使用情况
    • fd -h
