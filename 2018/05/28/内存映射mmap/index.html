内存映射mmap

字数统计: 3,332 | 阅读时长 ≈ 13

此篇博客的偏向点主要是对于内存映射的原理说明。

mmap的初识

mmap是一种内存映射文件的方法，即将一个文件或者其它对象映射到进程的地址空间，实现文件磁盘地址和进程虚拟地址空间中一段虚拟地址的一一对映关系,如果虚拟区域比文件区要大那么剩余的区域用0来填充。实现这样的映射关系后，进程就可以采用指针的方式读写操作这一段内存，而系统会自动回写脏页面到对应的文件磁盘上，即完成了对文件的操作而不必再调用read,write等系统调用函数。相反，内核空间对这段区域的修改也直接反映用户空间，从而可以实现不同进程间的文件共享。

linux虚拟内存区域

  • 在大概知道什么是mmap以后，想要知道mmap的底层实现原理，我们还需要对linux虚拟内存区域的管理有一个大致的认识。

Linux将虚拟内存组织成一些区域（也叫做段）的集合，一个区域就是已经存在着的（已经分配的）虚拟内存的连续片，这些页是以某种方式相关联的。每个存在的虚拟页都保存在某个区域当中，而不属于某个区域的虚拟页式不存在的。即虚拟地址空间中有效的虚拟页面都对应着相应的虚拟页，不在这个虚拟页面中的虚拟页式不存在，并且不能被进程引用，即只要你虚拟页不在我这个虚拟页面我就认为你不存在，这样的话内核也不会去记录不存在的虚拟页，而这样的页也不任何资源。

  • 在linux中内核为每个进程维护了一个task_struct结构体（PCB进程控制块）。这个控制块中的一些成员就是记录整个进程所需要的所有的信息，比如：PID，指向用户栈的指针，可执行目标文件的名字，以及程序计数器），那么管理虚拟地址空间的信息也在这个结构体中。

在task_struct结构体中有一个字段mm，其指向mm_struct（内存描述符），它描述了虚拟内存当前的状态。其中的pgd指向第一级页面的基地址，当内核运行这个进程的时候就将pgd放在cr3控制寄存器中。而mmap指向vm_area_struct的结构体的链表，这个结构体描述的是当前虚拟地址空间的一块区域。

  • 一个vm_area_struct结构体应该包含以下字段

    • vm_start:指向这个区域的起始处
    • vm_end:指向这个区域的结束处
    • vm_prot:描述这个区域内包含的所有页的读写许可权限（在中断的时候要用）
    • vm_flags:描述这个区域内的页面是与其他进程共享的，还是这个进程私有的（还描述了一些其他的信息）
    • vm_next:指向链表中的下一个区域结构。

      同时其包含一个vm_ops指针，其内部可引出所有针对这个区域可以使用的系统调用函数。这样，进程对某一虚拟内存区域的任何操作需要用要的信息，都可以从vm_area_struct中获得。mmap函数就是要创建一个新的vm_area_struct结构，并将其与文件的物理磁盘地址相连。

  • 补充一点，Linux缺页异常处理

当MMU在试图翻译某个虚拟地址A的时候，触发了一个缺页。这个异常导致控制转到内核的缺页处理程序，处理程序随后会进行如下的步骤

  1. 虚拟地址A合法吗（A这个虚拟地址在虚拟地址在vm_area_struct描述的结构体内吗）？为了判断这个情况，缺页处理程序搜索区域结构的链表，把A和进程维护的vm_area_struct中的vm_start和vm_end作比较。如果这个地址是不合法的，那么缺页异常处理程序就触发一个段错误，从而终止这个进程。因为一个进程可能会创建很多的新的虚拟内存的区域，所以顺序搜索的话效率将会很差，所以在实际中在链表中建立了树，并在这棵树上进行查找。
  2. 试图进行内存访问是否合法？换句话说，进程是否有读、写或者执行这个区域的页面内容的权限？比如对于只有读权限的页面进行写操作，或者用户模式对内核区域读取数据，如果是这个样子的话，那么会根据vm_area_struct里面的页面权限来判断，当然这个是建立在A合法的基础上,如果不合法则触发一个保护异常，从而终止这个进程。
  3. 当现在为止排除了段错误和保护异常这类的错误，内核就知道了这是由于对合法地址进行方法引起的正常缺页，即物理内存中没有对应的页面，那么此时内核就会选择一个牺牲页面，然后判断其是否被修改过，修改过的话则将其写入外存，然后换入新的页面并更新页表，这里的换入换出其实就是对页表的修改，当缺页处理程序返回的时候，CPU重新启动引起缺页的指令，这次这条指令将A发送给MMU的时候,MMU就能正确的翻译A，而不会在产生缺页中断了。

文件内容到物理内存（主存）的拷贝

进程发起对这片映射空间的访问，引发缺页异常，实现文件内容到物理内存（主存）的拷贝

  1. 进程的读或写操作访问虚拟地址空间这一段映射地址，通过查询页表，发现这一段地址并不在物理页面上。因为目前只建立了地址映射，真正的硬盘数据还没有拷贝到内存中，因此引发缺页异常。

  2. 缺页异常进行一系列判断，确定无非法操作后，内核发起请求调页过程。

  3. 调页过程先在交换缓存空间（swap cache）中寻找需要访问的内存页，如果没有则调用nopage函数把所缺的页从磁盘装入到主存中。

  4. 之后进程即可对这片主存进行读或者写的操作，如果写操作改变了其内容，一定时间后系统会自动回写脏页面到对应磁盘地址，也即完成了写入到文件的过程。


注意：修改过的脏页面并不会立即更新回文件中，而是有一段时间的延迟，可以调用msync()来强制同步, 这样所写的内容就能立即保存到文件里了。

  • 知识拓展：脏页

    脏页－linux内核中的概念，因为硬盘的读写速度远赶不上内存的速度，系统就把读写比较频繁的数据事先放到内存中，以提高读写速度，这就叫高速缓存，linux是以页作为高速缓存的单位，当进程修改了高速缓存里的数据时，该页就被内核标记为脏页，内核将会在合适的时间把脏页的数据写到磁盘中去，以保持高速缓存中的数据和磁盘中的数据是一致的。
